TicketCommandHandler.cs 
public class CreateTicketCommandHandler
  : IRequestHandler<CreateTicketCommand,
    Result<TicketDto>>
{
  public async Task<Result<TicketDto>>
    Handle(CreateTicketCommand request,
           CancellationToken ct)
  {
    var ticket = Ticket.Create(
      request.Title,
      request.Channel,
      request.CustomerId
    );

    await _repository.AddAsync(ticket, ct);
    await _unitOfWork.CommitAsync(ct);

    return Result.Success(
      _mapper.Map<TicketDto>(ticket)
    );
  }
}

Projeto em Destaque

Plataforma de Gestão de Demandas

Solução completa para centralizar e agilizar o atendimento multicanal de uma empresa.

Uma plataforma unificada que integra múltiplos canais de comunicação em um único sistema de tickets, atendendo tanto clientes externos quanto equipes internas. O sistema garante rastreabilidade completa, respostas ágeis e total transparência no fluxo de atendimento.

Padrões Aplicados

CQRS

Command Query Responsibility Segregation

Separação clara entre operações de leitura e escrita. Commands alteram estado, Queries apenas consultam.

DDD

Domain Driven Design

Modelagem centrada no domínio do negócio. Aggregates, Value Objects e Domain Events bem definidos.

Result

Result Pattern

Retorno explícito de sucesso ou falha sem exceptions para controle de fluxo, com mensagens de erro tipadas.

Clean

Clean Architecture

Dependências apontam para dentro. O domínio não depende de infraestrutura, garantindo testabilidade e flexibilidade.
